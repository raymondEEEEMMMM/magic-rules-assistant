# CloudBase 云函数部署指南

## 部署方式选择

| 方式 | 优点 | 缺点 |
|------|------|------|
| 事件云函数 (推荐) | 简单、稳定 | 无法自定义 HTTP 路由 |
| HTTP 云函数 | 可自定义 HTTP 路由 | 需要 bootstrap，配置复杂 |

推荐使用**事件云函数**，通过 MCP 工具或 API Gateway 调用。

## 部署步骤

### 1. 前提条件

- CloudBase 环境已创建
- MySQL 数据库已创建并导入数据
- 已安装 MCP 工具并完成登录和环境绑定

### 2. 部署命令

```bash
# 创建事件云函数
mcp__cloudbase__createFunction(
    func={
        "name": "mtgAsk",
        "runtime": "Python3.10",
        "handler": "index.main_handler",
        "type": "Event",
        "timeout": 60,
        "isWaitInstall": true
    },
    functionRootPath="functions/mtgAsk"
)
```

### 3. 测试调用

```bash
# 使用 MCP 工具调用
mcp__cloudbase__invokeFunction(
    name="mtgAsk",
    params={
        "httpMethod": "GET",
        "path": "/api/search",
        "queryString": "q=flying"
    }
)
```

## 关键配置

### 数据库连接

云函数无法访问内网 VPC，必须使用**外网地址**。

在 `functions/mtgAsk/backend/database.py` 中：

```python
if self.is_cloud_function:
    # 强制使用外网地址
    self.mysql_host = 'sh-cynosdbmysql-grp-5ydpqjru.sql.tencentcdb.com'
    self.mysql_port = 27987
```

### 入口文件

| 云函数类型 | 入口文件 | 说明 |
|-----------|----------|------|
| 事件云函数 | `index.py` | 直接处理请求，不需要 bootstrap |
| HTTP 云函数 | `scf_index.py` | 需要 scf_bootstrap 启动 HTTP 服务器 |

### scf_bootstrap (HTTP 云函数用)

```bash
#!/bin/bash
python3 -u index.py
```

## 常见问题

### 1. scf_bootstrap not found
- **原因**: HTTP 云函数缺少 bootstrap 文件
- **解决**: 保留 scf_bootstrap 文件，或改用事件云函数

### 2. Can't connect to MySQL 172.17.0.5
- **原因**: 云函数无法访问内网地址
- **解决**: 使用外网地址 `sh-cynosdbmysql-grp-5ydpqjru.sql.tencentcdb.com:27987`

### 3. Runtime 不支持修改
- **原因**: 云函数 Runtime 创建后不可修改
- **解决**: 删除重建函数

### 4. FUNCTION_EXECUTE_FAIL
- **原因**: 可能是 HTTP 触发器配置问题
- **解决**: 使用 invokeFunction 测试，或配置 API Gateway
