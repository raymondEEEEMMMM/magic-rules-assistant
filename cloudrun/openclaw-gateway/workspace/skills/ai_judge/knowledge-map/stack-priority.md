# 堆叠与优先权 (Stack & Priority)

> 规则章节: 116.x, 405.x, 115.x

## 优先权基础

**116.1** 获得优先权意味着玩家可以采取行动（施放咒语、启动式异能、执行特殊动作）。

**116.3b** 非主动玩家在主动玩家的战斗前行动步骤和战斗后行动步骤获得优先权。（这是主要的互动窗口）

### 优先权的流转

**116.3c** 堆叠清空且所有玩家连续让过时，当前步骤或阶段结束。

## 关键时机详解

### 1. 施放咒语的时机

| 阶段/步骤 | 谁获得优先权 | 重要行动 |
|-----------|--------------|----------|
| 开始阶段 | 主动玩家 | 重置永久物，抓牌 |
| 战斗前行动步骤 | 双方轮流 | **主要互动窗口** |
| 宣告攻击者步骤 | 主动玩家先 | 宣告攻击者后，优先权给主动玩家 |
| 宣告阻挡者步骤 | 主动玩家先 | 宣告阻挡者后，优先权给主动玩家 |
| 战斗伤害步骤 | 主动玩家先 | 先攻/连击特殊处理 |
| 战斗后行动步骤 | 双方轮流 | **主要互动窗口** |
| 结束步骤 | 主动玩家 | 清除伤害，触发「结束步骤」异能 |

### 2. 堆叠运作

**405.1** 当一个玩家获得优先权时，他可以将咒语或异能放入堆叠顶部。

**405.2** 堆叠的解析：从顶部开始，逐个结算（后进先出）。

### 3. 特殊动作 vs 堆叠

**115** 特殊动作**不使用堆叠**且**不能被响应**：
- 横置地产生法术力
- 翻面朝上（变身/显化）
- 支付强化费用
- 将牌面朝下的永久物翻面
- 持续操控鬼怪（如御灵科技）
- 取回已贴附的灵气/武具

## 复杂场景

### 场景1：在宣告攻击者前响应
- 你只能在**战斗前行动步骤**采取行动
- 一旦主动玩家宣告「进入宣告攻击者步骤」，轮到非主动玩家获得优先权时，攻击者已经宣告完毕

### 场景2：法术力池的消耗
**106.4** 法术力池在阶段和步骤结束时清空，且造成法术力灼伤。

**106.4a** 例外：如果是瞬间时机且玩家有优先权，可以在清空之前响应。

### 场景3：触发式异能和优先权
**116.5** 每次有物品放入堆叠后，主动玩家获得优先权。

**116.2a** 触发式异能进入堆叠后，在任何人获得优先权前，先检查所有状态动作。

### 场景4：回合结束时的响应
- 「结束步骤」异能触发后，主动玩家获得优先权
- 如果堆叠清空，阶段不会立即结束（因为有「结束步骤」异能）
- 双方可以在此步骤正常互动

## 常见误区

1. **「响应回合结束」**：
   - 没有「回合结束」这个动作
   - 你可以在结束步骤施放咒语，但这不是「响应」结束

2. **「响应攻击」**：
   - 只能在战斗前行动步骤采取行动
   - 一旦宣告攻击者，你就错过了时机

3. **法术力响应**：
   - 对手横置地产生法术力 - 这不是一个使用堆叠的动作，你不能响应
   - 你只能在对手获得优先权后响应他将要施放的咒语
