## 🗄️ 数据库设计

### SQLite 数据库（第一阶段）
- `rules` - 规则表
- `keyword_abilities` - 关键词异能表
- `card_rules` - 卡牌规则表
- `qa_templates` - 问答模板表

### 规则文件（第二阶段）
- **格式**: TXT（推荐，便于向量化）
- **大小**: 943 KB
- **内容**: 1133 条规则 + 190 个关键词异能

## 🚀 API 端点

### 基础查询
| 端点 | 方法 | 说明 |
|------|------|------|
| `/` | GET | 服务状态检查 |
| `/docs` | GET | Swagger API 文档 |
| `/wechat` | GET | 微信服务器验证 |
| `/wechat` | POST | 微信消息接收 |

### 规则查询
| 端点 | 方法 | 说明 |
|------|------|------|
| `/api/search?q=` | GET | 规则综合搜索 |
| `/api/keyword?k=` | GET | 关键词异能查询 |
| `/api/card?n=` | GET | 卡牌规则查询 |

### 规则管理
| 端点 | 方法 | 说明 |
|------|------|------|
| `/api/rules/update` | POST | 更新规则（可选force参数） |
| `/api/rules/parse` | GET | 解析规则文件 |
| `/api/rules/vectorization` | GET | 获取向量化数据 |
| `/api/rules/status` | GET | 获取规则状态 |

## 💻 技术栈

- **后端框架**: FastAPI 0.104.1
- **服务器**: Uvicorn 0.24.0
- **数据验证**: Pydantic 2.5.0
- **数据库**: SQLite
- **定时任务**: APScheduler 3.10.4
- **HTTP客户端**: requests
- **HTML解析**: BeautifulSoup4
- **配置管理**: python-dotenv


## 📌 关键决策

### 1. 格式选择：TXT格式 ✅

**原因：**
- 纯文本，向量化最友好
- 解析最简单，正则提取即可
- 结构保留完整，规则编号清晰
- 文件体积小（943 KB）
- 处理速度快

### 2. 定时任务：APScheduler ✅

**配置：**
- 每天03:00：检查更新
- 每周一10:00：全面检查
- 支持手动触发

### 3. 数据库：SQLite（初期）→ PostgreSQL（生产）✅

**原因：**
- SQLite：开发简单，无需额外配置
- PostgreSQL：生产环境，支持高并发
