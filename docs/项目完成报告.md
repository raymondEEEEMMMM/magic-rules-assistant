## 🗄️ 数据库设计

### 数据表结构

#### rules (规则表)
```sql
CREATE TABLE rules (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    rule_number TEXT UNIQUE NOT NULL,
    rule_title TEXT NOT NULL,
    rule_content TEXT NOT NULL,
    category TEXT,
    keywords TEXT
)
```

#### keyword_abilities (关键词异能表)
```sql
CREATE TABLE keyword_abilities (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    keyword_name TEXT UNIQUE NOT NULL,
    description TEXT NOT NULL,
    full_text TEXT NOT NULL,
    examples TEXT
)
```

#### card_rules (卡牌规则表)
```sql
CREATE TABLE card_rules (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    card_name TEXT UNIQUE NOT NULL,
    card_type TEXT,
    oracle_text TEXT NOT NULL,
    related_rules TEXT
)
```

#### qa_templates (问答模板表)
```sql
CREATE TABLE qa_templates (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    question TEXT NOT NULL,
    answer TEXT NOT NULL,
    keywords TEXT,
    category TEXT
)
```

## 🎯 使用示例

### 1. 启动服务
```bash
./start.sh
# 或
python3 backend/main.py
```

### 2. 测试 API
```bash
# 搜索规则
curl 'http://localhost:8000/api/search?q=飞行'

# 查询卡牌
curl 'http://localhost:8000/api/card?n=黑莲花'

# 查询异能
curl 'http://localhost:8000/api/keyword?k=践踏'
```

## 📝 注意事项

1. **微信配置**: 需要在微信公众平台配置服务器 URL 和 Token
2. **内网穿透**: 本地开发时可以使用 ngrok 等工具测试微信回调
3. **数据更新**: 规则数据需要定期更新以保持准确性
4. **生产部署**: 建议使用 PostgreSQL 或 MySQL 替代 SQLite
5. **安全性**: 生产环境需要添加 API 限流和访问控制
