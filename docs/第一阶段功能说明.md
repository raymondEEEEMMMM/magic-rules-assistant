```
backend/
├── main.py              # 服务入口
├── routes.py            # API路由
├── config.py            # 配置管理
├── database.py          # 数据库操作
├── init_data.py         # 数据初始化
├── services/
│   └── rule_service.py  # 规则服务逻辑
└── wechat/
    └── handlers.py      # 消息处理器
```

## 数据库结构

### rules (规则表)
- id: 主键
- rule_number: 规则编号
- rule_title: 规则标题
- rule_content: 规则内容
- category: 分类
- keywords: 关键词(JSON)

### keyword_abilities (关键词异能表)
- id: 主键
- keyword_name: 异能名称
- description: 简短描述
- full_text: 完整规则文本
- examples: 示例(JSON)

### card_rules (卡牌规则表)
- id: 主键
- card_name: 卡牌名称
- card_type: 卡牌类型
- oracle_text: 规则文本
- related_rules: 相关规则(JSON)

### qa_templates (问答模板表)
- id: 主键
- question: 问题
- answer: 答案
- keywords: 关键词(JSON)
- category: 分类

## API 端点

### 公开端点
- `GET /` - 服务状态
- `GET /wechat` - 微信服务器验证
- `POST /wechat` - 微信消息接收

### 调试端点
- `GET /api/search?q=` - 规则搜索
- `GET /api/keyword?k=` - 关键词异能查询
- `GET /api/card?n=` - 卡牌规则查询

## 使用示例

### 1. 启动服务
```bash
cd backend
python main.py
```

### 2. 初始化数据
```bash
cd backend
python init_data.py
```

### 3. 配置微信
1. 复制 `.env.example` 为 `.env`
2. 填入微信公众号配置信息
3. 在微信公众平台配置服务器URL

### 4. 测试API
```bash
# 搜索规则
curl "http://localhost:8000/api/search?q=飞行"

# 查询卡牌
curl "http://localhost:8000/api/card?n=黑莲花"

# 查询关键词
curl "http://localhost:8000/api/keyword?k=践踏"
```

```
用户: 飞行

      702.9. 飞行
      702.9a 飞行是一种静态异能，影响生物进行阻挡的方式。
      ...

用户: 卡牌:黑莲花

      类型: 神器

      规则文本:
      牺牲黑莲花：加三点任意颜色的法术力。

      相关规则: 106. 法术力, 701. 牺牲

系统: 🃏 万智牌规则助手使用指南

      • 异能:异能名称 - 快速查询异能

      • /start - 显示欢迎信息

      ...
```
