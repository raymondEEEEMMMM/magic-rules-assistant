# mtgAsk 小程序交互设计方案

## 1. 产品概述

**产品名称**: mtgAsk
**产品定位**: 万智牌规则问答助手
**目标用户**: 万智牌玩家、裁判、规则爱好者
**核心功能**: 卡牌查询、关键词检索、规则搜索、AI裁判问答

---

## 2. 信息架构

```
mtgAsk
├── 首页 (index)
│   ├── Banner + 主题切换
│   ├── 搜索入口 → 搜索页

│   ├── 欢迎消息 + 示例问题
```

## 3. 页面交互设计

### 3.1 首页 (index)

**功能**: 产品入口，展示核心功能入口

**布局元素**:
| 元素 | 描述 | 交互 |
|------|------|------|
| Banner | 渐变背景 + 产品标题 | - |
| 主题切换按钮 | ☀️/🌙 图标 | 点击切换日/夜间主题 |
| 搜索框 | 白色圆角输入框 | 点击跳转搜索页 |
| AI裁判卡片 | 渐变紫色背景 + 天平图标 | 点击跳转AI裁判页 |

**主题切换**:
- 默认日间主题（浅色背景）
- 点击右上角 ☀️ 切换为夜间主题
- 主题偏好保存到本地存储

---

### 3.2 搜索页 (search)

**功能**: 统一搜索入口，支持关键词/卡牌/规则搜索

**布局元素**:
| 元素 | 描述 | 交互 |
|------|------|------|
| 搜索框 | 顶部输入框 | 输入关键词，自动显示清空按钮 |
| 类型切换 | 全部/关键词/卡牌/规则 Tab | 点击切换搜索类型 |
| 搜索历史 | 关键词标签列表 | 点击使用历史关键词搜索 |
| 结果列表 | 分组展示不同类型结果 | 点击跳转详情页 |

**搜索逻辑**:
1. **全部模式**: 按关键词→卡牌→规则顺序查询，遇结果即停
2. **单一类型模式**: 只查询选定类型

**结果穿透**:
- 关键词结果 → 关键词详情页
- 卡牌结果 → 卡牌详情页
- 规则结果 → 规则详情页

---

### 3.3 AI裁判页 (agent)

**功能**: AI对话式规则问答

**布局元素**:
| 元素 | 描述 | 交互 |
|------|------|------|
| 导航栏 | 返回、标题、清空按钮 | 点击清空会话历史 |
| 欢迎消息 | AI气泡 + 示例问题卡片 | 点击示例直接发送 |
| 消息列表 | 用户/AI消息气泡 | 展示对话内容 |
| 输入区域 | 输入框 + 发送按钮 | 输入问题，点击发送 |

**消息气泡样式**:
| 类型 | 样式 |
|------|------|
| 用户消息 | 紫色渐变背景，右对齐 |
| AI消息 | 白色背景，左对齐，带阴影 |
| Loading | 白色气泡 + "思考中" + 跳动圆点 |

**Markdown渲染**: AI回复内容支持Markdown格式（标题、加粗、列表等）

**主题切换**: 导航栏右侧 ☀️/🌙 按钮，点击切换

---

### 3.4 卡牌页 (card)

**功能**: 卡牌搜索和详情查看

**布局元素**:
| 元素 | 描述 | 交互 |
|------|------|------|
| 搜索框 | 顶部输入框 | 输入卡名搜索 |
| 卡牌列表 | 图片 + 名称 + 类型 | 点击查看详情 |
| 无结果 | 空状态提示 | - |
| 加载中 | 旋转圆点动画 | - |

**卡牌详情（全屏页面）**:
| 区域 | 内容 |
|------|------|
| 返回按钮 | 顶部固定，返回上一页 |
| 卡牌图片 | 居中展示，带阴影 |
| 基础信息 | 名称（中英文）、法力费用、类型、力量/防御、所属系列 |
| 关键词异能 | 紫色标签展示 |
| Oracle文本 | 中文规则文本，支持换行 |
| 英文Oracle | 英文规则文本（与中文分开显示） |

**换行符处理**: API返回的 `\n` 字符转换为真正换行

---

### 3.5 关键词页 (keyword)

**功能**: 关键词异能搜索和详情

**布局元素**:
| 元素 | 描述 | 交互 |
|------|------|------|
| 搜索框 | 顶部输入框 | 输入关键词搜索 |
| 常用关键词 | 标签网格 | 点击直接查看详情 |
| 搜索结果 | 关键词列表 | 点击查看详情 |
| 关键词详情 | 中英文名称 + 描述 + 相关规则 | - |

---

### 3.6 规则页 (rule)

**功能**: 规则详情查看

**布局元素**:
| 元素 | 描述 | 交互 |
|------|------|------|
| 返回按钮 | 顶部 | 返回上一页 |
| 语言切换 | CN/EN 切换 | 切换中英文规则显示 |
| 规则编号 | 大号显示 | - |
| 规则内容 | 按规则编号分段显示 | 每条规则独立段落 |

**编号分段规则**: 按 `XXX.Y` 或 `XXX.Ya` 格式分割，保证编号完整

## 4. 主题系统

### 4.1 日间主题（默认）
| 元素 | 颜色 |
|------|------|
| 背景 | #f5f5f5 |
| 卡片背景 | #ffffff |
| 主文字 | #333333 |
| 次要文字 | #666666 |
| 强调色 | #667eea / #764ba2 |
| 输入框背景 | #f0f0f0 |

### 4.2 夜间主题
| 元素 | 颜色 |
|------|------|
| 背景 | #16213e |
| 卡片背景 | #1a1a2e |
| 主文字 | #ffffff |
| 次要文字 | rgba(255,255,255,0.6) |
| 强调色 | #ffd700 |
| 输入框背景 | rgba(255,255,255,0.1) |

### 4.3 主题切换机制
1. 全局状态存储在 `app.globalData.isLightTheme`
2. 各页面 `onShow` 时同步全局主题状态
3. 切换时更新所有已打开页面的样式
4. 主题偏好持久化到 `wx.setStorageSync('appTheme')`

## 5. API接口设计

### 5.1 搜索接口
```
GET /api/search?q={keyword}
```
- 返回: `{ keywords: [], cards: [], rules: [] }`

### 5.2 关键词接口
```
GET /api/keyword?k={keyword}
```
- 返回: `{ result: { keyword_name, keyword_en, description, rule_ref } }`

### 5.3 卡牌搜索
```
GET /api/card?q={cardName}&page=1&pageSize=10
```
- 返回: `{ count, items: [...] }`

### 5.4 卡牌详情
```
GET /api/card?id={cardId}
```
- 返回: 完整卡牌对象

### 5.5 规则搜索
```
GET /api/rules?q={keyword}
```
- 返回: `{ results: { rules: [...] } }`

### 5.6 规则详情
```
GET /api/rule?num={ruleNumber}
```
- 返回: 完整规则内容

### 5.7 AI裁判
```
POST /api/ai-judge/chat
Body: { message, session_id }
```
- 返回: `{ success, reply }`
